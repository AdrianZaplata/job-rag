<objective>
Fix 1 of 5 (Phase 04.1) — Move the `users.entra_oid` UPDATE from
`alembic/versions/0005_adopt_entra_oid.py::upgrade()` into a new helper called
from `init_db()` in `src/job_rag/db/engine.py`, so the UPDATE re-runs on every
container boot (idempotent post-alembic step) instead of only once per
revision-marker.

**Why:** Alembic's `upgrade()` runs once per revision-marker — after the first
container boot landed `0005` at head, every subsequent restart skips the UPDATE
entirely. Rotating `SEEDED_USER_ENTRA_OID` currently requires manual
`az containerapp exec --command python3` running inline SQLAlchemy (per Phase 4
close-out deviation #3 + memory `alembic-update-runs-once.md`). After this fix,
rotation only needs an ACA revision restart.

**Source of truth for this deviation:**
`.planning/phases/04-frontend-shell-auth/04-06-SUMMARY.md` deviation #3
(Auto-fixed Issues section — "Migration 0005's UPDATE only fires once per
revision via Alembic").

**Output:** Code change + test coverage + one atomic commit. No remote push,
no cloud touch.
</objective>

  def test_seed_entra_oid_no_op_when_env_unset(self, monkeypatch):

  def test_seed_entra_oid_executes_update_when_env_set(self, monkeypatch):
      """_seed_entra_oid runs UPDATE with bind params when SEEDED_USER_ENTRA_OID is set."""
      test_oid = "18d774c1-62ac-4416-8945-b5eca715e9ed"
      monkeypatch.setenv("SEEDED_USER_ENTRA_OID", test_oid)

      mock_engine = MagicMock()
      mock_engine.begin.return_value.__enter__.return_value = mock_conn
      monkeypatch.setattr("job_rag.db.engine.engine", mock_engine)

      from job_rag.db.engine import _seed_entra_oid
      _seed_entra_oid()

      mock_engine.begin.assert_called_once()
      mock_conn.execute.assert_called_once()
      # Inspect the bind params on the executed TextClause
      executed_stmt = mock_conn.execute.call_args[0][0]
      compiled = executed_stmt.compile()
      assert "UPDATE users SET entra_oid" in str(compiled)
      # Bind params present
      assert executed_stmt.compile().params.get("oid") == test_oid
      assert executed_stmt.compile().params.get("seeded_uuid") == "00000000-0000-0000-0000-000000000001"
