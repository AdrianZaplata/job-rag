<objective>
Harden the 4 out-of-band-seeded KV secret resources in `infra/envs/prod/main.tf` so that
`value_wo_version` (the trigger that tells AzureRM to re-write `value_wo`) is included in
each resource's `lifecycle.ignore_changes` list. This prevents the Phase 06 UAT regression
(Bugs #1 + #2 in `06-UAT-DEBUG-HANDOFF.md`) where `az keyvault secret set`-seeded real
OpenAI / Langfuse / seeded-user-oid values were silently wiped back to `"managed-out-of-band"`
on a subsequent `terraform apply`.

Purpose: An out-of-band-seeded secret must survive every future apply. `[value]` alone does
NOT protect `value_wo` — confirmed by repo memory `terraform-value-wo-overwrites-secret.md`.

Output: 3 modified lifecycle blocks + 1 added lifecycle block in `infra/envs/prod/main.tf`.
A `terraform plan` against the live state (run by Plan 04) shows ZERO diff for these 4
resources.
</objective>

- `value_wo_version` (integer trigger: bumping it tells provider to push `value_wo` again)
- `ignore_changes = [value]` does NOT cover `value_wo` or `value_wo_version`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TF state ↔ live Azure KV | Out-of-band-seeded secret values cross this boundary; TF state holds only placeholders + `value_wo_version` trigger |
| Out-of-band `az keyvault secret set` ↔ ACA `secretRef` resolution | Live secret value reaches ACA env vars at revision creation; the ignore_changes block protects this seeded value from being overwritten by future `terraform apply` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06.1-01 | Tampering | KV secret value | mitigate | `lifecycle.ignore_changes = [value, value_wo, value_wo_version]` on all 4 KV secret resources prevents `terraform apply` from clobbering out-of-band-seeded real values (this PR's entire purpose). Reduces risk vs. the pre-PR `[value]`-only state. |
| T-06.1-02 | Information Disclosure | TF state file | accept | TF state holds only the placeholder string `"managed-out-of-band"` (or `var.seeded_user_entra_oid` which is itself non-secret on first apply per Phase 3 D-09); the real OpenAI/Langfuse secrets never enter state. No new exposure. Preserves Phase 3 D-13 KV-vs-plain-env model. |
| T-06.1-03 | Denial of Service | ACA service availability | mitigate | The pre-PR regression caused ACA cold-starts to crash on missing OPENAI_API_KEY after a `terraform apply` re-wrote the placeholder. This PR eliminates that failure mode. |
