# Phase 06.1 — Security

> Per-phase security contract: threat register, accepted risks, and audit trail.
>
> Scope: defensive infra-only phase that closes three Phase 06 UAT regressions
> (KV `value_wo` placeholder re-write wiping out-of-band-seeded secrets;
> hardcoded ACA cpu/memory reverting Adrian's manual OOM-fix bump; runbook's
> single-var-file `terraform apply` silently wiping Entra auth env vars).
> No new runtime attack surface; all threats are infra/operational supply-chain
> integrity (Tampering, Information Disclosure, Denial of Service, Repudiation).

---

## Trust Boundaries

Consolidated across all 4 plans (01 = KV lifecycle, 02 = compute size, 03 =
runbook + scripts + CI, 04 = verification gate).

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| TF state ↔ live Azure KV | Out-of-band-seeded secret values live only in Azure KV; TF state holds the `"managed-out-of-band"` placeholder + `value_wo_version` trigger | Placeholder string (non-secret); real secrets never enter TF state |
| Out-of-band `az keyvault secret set` ↔ ACA `secretRef` resolution | Live secret value reaches ACA env vars at revision creation time | OPENAI_API_KEY, LANGFUSE_*, SEEDED_USER_ENTRA_OID (sensitive) |
| Repo (PR-visible `prod.tfvars`) ↔ live ACA container template | Container resource sizing (cpu, memory) crosses on every `terraform apply` | Non-sensitive deployment config (cpu=1.0, memory=2Gi) |
| Runbook prose ↔ operator-typed shell command | Operator copies+executes terraform commands documented in README | Var-file flag composition (the Bug #3 root-cause surface) |
| `scripts/tf-apply-prod.sh` wrapper exit code ↔ operator decision | Missing `prod.tfvars.local` → wrapper exits 2 with remediation | None — file presence check only, no contents read |
| CI runner ↔ live ACA container template | CI-driven `terraform apply` (GitHub Actions) without `prod.tfvars.local` could silently wipe auth env vars | Pre-apply guard reads `properties.template.containers[0].env` from live ACA to assert non-empty before applying |
| Adrian's local `az login` ↔ workforce-tenant prod subscription | Verification commands (`terraform plan`, `az keyvault secret show`) read live state | Read-only — no apply during verification |
| Repo (`prod.tfvars.local`, gitignored) ↔ operator workstation only | Local-only auth values (`backend_audience`, `entra_tenant_id`, `entra_tenant_subdomain`, `home_ip`, `api_audience`) never traverse the repo or GitHub Actions | Public-by-design auth values + home IP (low-sensitivity per Phase 3 D-13) |
